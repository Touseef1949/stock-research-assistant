# Security policy

## Reporting a vulnerability

Do not disclose a suspected vulnerability in a public issue. Use GitHub's
private vulnerability reporting for this repository, or email
`tshaik1990@gmail.com` with the subject `Stock Research Assistant security report`.

Include the affected revision, reproduction steps, impact, and suggested
mitigation. You can expect acknowledgement within seven days. Do not attach
broker credentials, portfolio data, payment records, personal data, API keys,
or unsanitized reports.

## Supported versions

Only the latest tagged release and current `main` branch receive security
updates.

## Financial and AI boundary

This software produces educational research, not investment advice. It does
not place orders. See [docs/THREAT_MODEL.md](docs/THREAT_MODEL.md) for the model,
market-data, broker, authentication, and report-data boundaries.
